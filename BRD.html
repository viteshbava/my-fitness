Business Requirements Document (BRD)

Fitness Training App

Document Version: 1.0
Date Created: January 2, 2026
Last Updated: January 5, 2026
Status: In Progress

Document Purpose

This document outlines the functional requirements for a personal fitness training application. The purpose is to establish a clear understanding of what features and capabilities the app needs before development begins.

Functional Requirements

1. Exercise Management

1.1 Exercise Database

The application will maintain a database of exercises with the following attributes:

  • Name: The exercise name
  • Video URL Link: A Google Drive link to an instructional video
  • Movement Type: Classification of the movement
  • Pattern: The movement pattern category
  • Primary Body Part: The main body part targeted by the exercise
  • Secondary Body Part: Secondary body parts engaged during the exercise
  • Equipment: Required equipment for the exercise
  • Is Mastered: Boolean flag indicating whether the exercise has been learned/mastered
  • Notes: Free-text field for additional information or instructions about the exercise ADDED

1.2 Data Migration

Initial exercise data will be imported via CSV file upload.

1.3 CRUD Operations

Phase 1 will support Read-only operations for exercises., with the exception of the Notes field which will be editable. Full CRUD (Create, Read, Update, Delete) functionality will be considered for future phases. MODIFIED

1.4 Exercise Library View ADDED

The application will provide a library view to browse and access all exercises in the system.

1.4.1 Search Functionality
  • Users can search exercises by exercise name
1.4.2 Filter Functionality

Users can filter the exercise library by the following criteria:

  • Movement Type
  • Pattern
  • Primary Body Part
  • Secondary Body Part
  • Equipment
  • Is Mastered (Yes/No)
1.4.3 Sorting Functionality

Users can sort exercises by:

  • Alphabetical by Name (default)
  • Last Used Date

1.5 Exercise Detail View ADDED

When a user selects an exercise from the library, a detailed view will be displayed with the following information and capabilities:

1.5.1 Exercise Attributes Display
  • Display all exercise attributes (Name, Video URL Link, Movement Type, Pattern, Primary Body Part, Secondary Body Part, Equipment, Is Mastered) in read-only mode
  • The Notes field will be editable, allowing users to update equipment settings and other relevant information
1.5.2 Video Display
  • Display a video thumbnail when a Video URL Link is present
  • When the thumbnail is clicked, display the playable video in a modal window
  • When no Video URL Link is present, display "No video available" text
1.5.3 Historical Performance Data

Display a table showing all historical workout data for the exercise, grouped by session:

  • Columns: Date, Set Number, Reps, Weight
  • Each session groups all sets performed on that date
1.5.4 Maximum Weight Score
  • Display the single heaviest weight where the user completed at least 6 reps in any set
  • This serves as the current "score" or personal record for the exercise
1.5.5 Progress Chart

Display a line chart showing progression over time:

  • X-axis: Actual dates
  • Y-axes: Two Y-axes (left and right) for Weight and Reps, color-coded
  • Data range: Last 12 months of historical data
  • Chart type: Line chart with two color-coded lines (one for weight, one for reps)
